# ChatGPT 对话寻踪

一个完全运行在浏览器本地的 Chrome / Edge 扩展。它会实时索引你在
`chatgpt.com` 中实际打开的对话，并提供中文关键词搜索、上下文预览和消息定位。

> 本项目不是 OpenAI 官方产品，也不隶属于 OpenAI。

## 能做什么

- 自动收录安装后打开或新产生的 ChatGPT 对话
- 流式回复完成前持续更新，不会重复保存半截回答
- 多关键词搜索，例如 `浏览器 本地 索引`
- 显示命中消息、对话标题和更新时间
- 点击结果后打开对应对话，并滚动到命中位置
- 所有索引保存在浏览器扩展的本地存储中
- 支持把全部索引备份为 JSON，并在重装或换电脑后恢复

## 安装

### Chrome

1. 地址栏打开 `chrome://extensions/`
2. 打开右上角的“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本文件夹：`chatgpt-live-search-extension`
5. 打开或刷新 `https://chatgpt.com/`

### Edge

1. 地址栏打开 `edge://extensions/`
2. 打开左侧的“开发人员模式”
3. 点击“加载解压缩的扩展”
4. 选择本文件夹：`chatgpt-live-search-extension`
5. 打开或刷新 `https://chatgpt.com/`

## 使用

- 打开任意一段对话，它会自动加入索引。
- 点击页面右下角的绿色按钮，或按 `Alt+K` 打开搜索面板。
- 多个关键词用空格分隔；结果需要同时包含这些关键词。
- 点击浏览器工具栏中的扩展图标也可以开关面板。
- 面板底部可以清空本地索引。该操作不会删除 ChatGPT 原始对话。
- 面板底部的“备份”会下载完整索引；“恢复”可以合并导入该 JSON。

## 长期记忆如何工作

- 每次打开一段对话，扩展都会以 ChatGPT 对话 ID 为主键更新本地记录。
- 关闭网页或重启 Edge 不会清除记录，之后仍然可以直接搜索。
- 重新加载同一扩展目录不会清除记录。
- 卸载扩展、清除 Edge 扩展数据或更换浏览器配置会删除本地记录，因此建议定期点击“备份”。
- 恢复备份时会按对话 ID 合并；同一段对话保留更新时间较新的版本。

## 隐私与边界

- 扩展不会读取、保存或发送登录令牌。
- 扩展不调用 ChatGPT 未公开接口。
- 扩展没有服务器，也不会上传对话内容。
- 它只索引安装后在页面中实际渲染过的对话；没有打开过的旧对话不会凭空出现。
- 本地记忆会持续存在，直到你清空索引、卸载扩展或清除该扩展的浏览器数据。
- 如果 ChatGPT 大幅调整消息节点结构，采集选择器可能需要随版本更新。

## 验证

项目不需要安装依赖。安装了 Node.js 时，可在本目录运行：

```powershell
npm test
npm run check
```

## 项目结构

```text
.
├─ manifest.json          # Manifest V3 配置
├─ background.js          # 工具栏按钮
├─ content.js             # 采集、存储、搜索面板和备份恢复
├─ search-core.js         # 可独立测试的搜索逻辑
├─ panel.css              # Shadow DOM 面板样式
├─ tests/                 # 无依赖测试
└─ PRIVACY.md             # 隐私说明
```

## 开发

修改源码后，在 `edge://extensions/` 或 `chrome://extensions/` 中点击扩展卡片上的
“重新加载”，再刷新 ChatGPT 页面即可。
